# Phase 3 Plan 1: App Shell, Routing, and Auth Gate

## Objective

Set up shadcn/ui, React Router, authentication context with shared password, and the app shell layout (header + main content area). After this plan, the app has routing, a login gate, and a consistent shell for all pages.

## Context

- @src/App.tsx - Current placeholder app with QueryClientProvider
- @src/main.tsx - Entry point
- @src/lib/config.ts - Project registry and config
- @package.json - Dependencies (react-router-dom already installed)

## Tasks

### Task 1: Initialize shadcn/ui and install required components
type="auto"

Install shadcn/ui CLI, initialize it, and add the Card and Button components needed for the auth page and dashboard.

**Done when:**
- components.json exists at project root
- shadcn/ui utility `cn()` exists at `src/lib/utils.ts`
- Card and Button components exist in `src/components/ui/`
- `globals.css` has shadcn/ui CSS variables and Tailwind imports

### Task 2: Auth context and login page
type="auto"

Create a shared-password auth system using React Context + sessionStorage. The password is compared against `VITE_AUTH_PASSWORD` env var.

**Files:**
- `src/contexts/AuthContext.tsx` - AuthProvider, useAuth hook
- `src/pages/LoginPage.tsx` - Simple login form

**Done when:**
- AuthContext provides `isAuthenticated`, `login(password)`, `logout()`
- Login persists to sessionStorage
- LoginPage renders a password input and submit button
- Wrong password shows an error message

### Task 3: React Router setup with auth-protected routes
type="auto"

Set up React Router with a BrowserRouter, route definitions, and a ProtectedRoute wrapper that redirects to login if not authenticated.

**Files:**
- `src/components/ProtectedRoute.tsx`
- `src/App.tsx` - Updated with Router and routes

**Done when:**
- `/` route shows overview page (placeholder for now)
- `/project/:id` route shows project detail page (placeholder for now)
- `/login` route shows LoginPage
- Unauthenticated users are redirected to `/login`

### Task 4: App shell layout with header
type="auto"

Create the app shell layout with a header showing the dashboard title, a last-updated timestamp area, and a refresh button placeholder. The layout wraps all authenticated pages.

**Files:**
- `src/components/layout/AppLayout.tsx` - Header + main content area
- `src/pages/OverviewPage.tsx` - Placeholder overview page
- `src/pages/ProjectDetailPage.tsx` - Placeholder detail page

**Done when:**
- AppLayout renders a header with "Dashboard Tech Projects" title
- Header has a placeholder area for last-updated timestamp
- Header has a placeholder refresh button
- Main content area renders child routes via Outlet
- Both placeholder pages render inside the layout

## Verification

```bash
npm run build
npm run test
```

## Success Criteria

- shadcn/ui is initialized and Card/Button components are available
- Auth gate blocks unauthenticated access
- React Router navigates between login, overview, and detail pages
- App shell provides consistent header layout
- Build succeeds with zero errors
